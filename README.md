# Серверное приложение информационной системы NeuroNotes 

Данный репозиторий содержит исходный код серверной части информационной системы интеллектуальной обработки аудио-текстовых данных, разработанной в рамках выпускной квалификационной работы. Исходный код клиентского веб-приложения представлен в связанном репозитории [neuro-notes-web-client](https://github.com/Fairlier/neuro-notes-web-client). Программный комплекс автоматизирует цикл работы с персональными записями. Разработанная система выполняет преобразование речи в текст, автоматическую тематическую классификацию заметок, формирование их структурированного представления и краткого резюме, а также осуществляет семантический и диалоговый доступ к информации с применением механизма генерации ответов с дополненной выборкой (RAG). 

Архитектура серверного приложения спроектирована в соответствии с принципами предметно-ориентированного проектирования и чистой архитектуры. В основе программного интерфейса лежит принцип разделения ответственности команд и запросов (CQRS). Реализация выполнена на базе платформы ASP.NET Core 8. В качестве основной системы управления базами данных задействована PostgreSQL с векторным расширением pgvector. Для хранения бинарных аудиофайлов интегрировано локальное объектное хранилище MinIO. Выполнение ресурсоемких задач, включая обращения к нейросетевым моделям и транскрибацию, вынесено в подсистему фоновой асинхронной обработки на базе библиотеки Hangfire. 

Интеллектуальная обработка данных построена на гибридном подходе с использованием локальных и облачных моделей машинного обучения. Локальная транскрибация речи реализована посредством акустических моделей Vosk. Тематическая классификация текстов выполняется с помощью дообученной дистиллированной многоязычной модели семейства BERT (DistilBERT), экспортированной в формат ONNX и запускаемой через среду ONNX Runtime. Взаимодействие с большими языковыми моделями абстрагировано и позволяет подключать локальные генеративные модели через платформу Ollama, а также внешние коммерческие программные интерфейсы Gemini и Mistral.

## Развертывание и запуск

Развертывание серверной части возможно как из исходного кода репозитория, так и в составе объединённого релиза всей системы.

**Развертывание и запуск из исходного кода.**  
Для запуска серверного приложения из исходного кода необходимо предварительно создать файл переменных окружения `.env` в корневой директории проекта на основе шаблона `.env.example`. Перед запуском требуется внимательно проверить и при необходимости изменить параметры окружения, включая настройки API, базы данных PostgreSQL, хранилища MinIO, JWT-аутентификации, провайдеров искусственного интеллекта, значения по умолчанию для пользовательских настроек, а также имена локальных моделей. Особое внимание следует уделить конфигурации моделей Vosk: они должны быть предварительно загружены и размещены в директории `NeuroNotes.Infrastructure/Resources/Models/Vosk` в соответствии с ожидаемыми именами каталогов, указанными в `.env`. При использовании контейнерного сценария пользователь не должен изменять файл `docker-compose.yml`, так как все параметры конфигурации выносятся в файл `.env`. Запуск серверного приложения совместно с инфраструктурными зависимостями выполняется в корневой директории проекта командой:

```
docker compose up -d
```

При использовании локальных моделей Ollama их загрузка в контейнерную среду выполняется отдельной командой:

```
docker compose --profile init up ollama-init
```

**Развертывание и запуск в составе объединённого релиза.**  
Для запуска готовой сборки всей системы в разделе релизов предоставляются объединённые файлы `docker-compose.yml` и `.env`. Данный вариант развертывания предназначен для одновременного запуска клиентского приложения, серверной части и всех необходимых инфраструктурных сервисов. Пользователь должен изменять только файл `.env`, внимательно проверяя сетевые параметры, значения портов, настройки по умолчанию, имена локальных моделей и иные параметры окружения. Файл `docker-compose.yml` редактировать не требуется. Для корректной работы локального распознавания речи модели Vosk должны быть заранее размещены в директории `./Models/Vosk` относительно каталога, содержащего файлы развертывания. Запуск системы выполняется командой:

```
docker compose up -d
```

При использовании локальных моделей Ollama их загрузка в контейнерную среду выполняется отдельной командой:

```
docker compose --profile init up ollama-init
```
